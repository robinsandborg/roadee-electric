# Product Feedback Community Platform - Frame

## Source

> We're shaping a product idea for a website that lets you, as a product company, interact with your users to get feedback and feature requests.
>
> The solution should be community-led, letting users interact with each other, giving feedback and support for feature requests.
>
> The product should let you sign up and create a space for your product. Here you should be able to add colleagues. Normal users should be able to sign up with a social sign in.
>
> Staff can have titles and profiles visible to users, so that a users can clearly see when they are interacting with a user, and when they are interacting with staff
>
> User should be able to create and comment on posts, as well as upvote others and edit own posts.
>
> Staff should be able to delete posts, and delete comments
>
> When a suggestion is deemed ready for work, staff can clarify/describe requests in a way where the users original is preserved - maybe creating a document above the thread which is the businesses understanding of the feature
>
> A post should be able to go from suggestion -> selected for development -> beta availability -> GA. A post should be able to have an image and rich text.
>
> All posts should have categories and tags. Categories can be things like "bug" "feature" "question" etc. Tags are used for things specific to the prodct. If you are making an instagram clone it could be things like "feed" "messages" "search" and so on.

---

## Problem

Product teams need a shared place to collect user feedback and requests, but existing ad hoc channels scatter context and make prioritization opaque. Users cannot easily collaborate around requests, and staff/user identity is often unclear, which weakens trust and support quality. Teams also lack a structured path from idea to release, with preserved user intent and clear lifecycle visibility.

## Outcome

A multi-tenant community feedback platform where product companies run dedicated spaces, users and staff collaborate transparently, requests move through a visible lifecycle, and staff can formalize selected ideas without losing original user context.
